# Release Notes: v2.0.0

`v2.0.0` is the first release where the public documentation is aligned to the current server surface.

---

## Release Surface

- **20 MCP tools** across retrieval, session, artifacts, entity, governance, hygiene, federation, and topology domains
- **4 MCP prompts**: `analysis_routing`, `memory_review`, `impact_before_edit`, `federated_sync`
- **2 MCP resources** (`graphbase://schema`, `graphbase://services`) and **2 resource templates** (`graphbase://health/{workspace_id}`, `graphbase://session/{session_id}`)
- **HTTP devtools** with token-gated write routes and confirm-gated write tool invocation

---

## Documentation Corrections

This release updates the docs to match the source of truth in `src/graphbase_memories/`:

- Removed the obsolete `route_analysis` tool from public tool inventories
- Corrected the registered tool count from `21` to `20`
- Added prompt/resource coverage where the docs previously only described tools
- Updated devtools route docs to include the current `X-Devtools-Token` and `confirm=true` requirements
- Refreshed schema and engine docs to match the current v3 graph model and engine layout

---

## Upgrade Notes

- Pin MCP host installs to `@v2.0.0` for a stable release target
- If you built workflows around `route_analysis`, migrate them to the `analysis_routing` prompt
- If you call devtools write routes directly, ensure you send `X-Devtools-Token` and, for destructive write-tool invokes, `confirm: true`

---
